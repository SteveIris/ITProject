### [ Сценарий 1 - Регистрация пользователя ]

1. Пользователь придумывает и вводит свой логин.
2. Пользователь придумывает и вводит свой пароль.
3. Система проверяет совпадение логина с уже имеющимся. Если такой логин уже существует, система выдаёт об этом сообщение и сценарий начинается заново.
4. Пользователь вводит свою электронную почту.
5. Если электронная почта соответствует формату и существует, система отправляет на неё письмо для подтверждения почты.
6. Пользователь подтверждает почту.
7. Запускается сценарий 2 - настройка приложения.

### [ Сценарий 2 - Настройка приложения ]

1. Система выводит на экран текст с объяснением принципов работы приложения и рекомендациями по его использованию для корректной работы.
2. Пользователь знакомится с текстом и нажимает на кнопку для продолжения.
3. Система просит у пользователя разрешение для доступа к камере устройства.
4. Пользователь реагирует на просьбу доступа к камере.
5. В случае отрицательного ответа пользователя система выводит на экран сообщение об ограниченности функционала приложения без доступа к камере и второй раз спрашивает разрешение.
6. Пользователь указывает в чекбоксе, хочет ли он получать уведомления об обновлениях приложения.
7. По желанию пользователь может указать толщину линии совего маркера\ручки\карандаша для ускорения и улучшения точности работы программы в дальнейшем.
8. Пользователь переходит в основное меню.

### [ Сценарий 3 - Получение изображения для игры ]

1. Пользователь в главном меню выбирает элемент "Играть".
2. Система выводит на экран три уровня сложности: Тренировочный, средний и сложный. Также система сообщает, что результаты раунда на тренировочном уровне не учитываются в статистике, а сложный и средний учитываются, причём сложный с большим коэффициентом.
3. Пользователь выбирает уровень сложности.
4. В зависимости от уровня сложности система выводит на экран чёрно-белое изображение c различными геометрическими фигурами.
5. Пользователь внимательно изучает и запоминает изображение.
6. Через 10 секунд после вывода на экран изображения оно пропадает, и пользователю предлагается воспроизвести его по памяти на экране устройства или на бумаге.
7. В зависимости от выбора пользователя запускается либо сценарий 4, либо сценарий 5.

### [ Сценарий 4 - Рисование на экране ]

1. Система полностью очищает экран, оставляя только белое пространство, на котором можно рисовать. Также система предлагает два инструмента, которые можно сменять на протяжении рисования - карандаш и ластик. Также на экране присутствует кнопка окончания ввода.
2. Пользователь, используя доступные инструменты воспроизводит ранее данное ему изображение на экране.
3. Закончив, пользователь нажимает на кнопку завершения рисования.
4. Система спрашивает у пользователя, действительно ли он хочет завершить рисование.
5. Если ответ пользователя отрицательный, сценарий возвращается к Шагу 2. В противном случае система обрабатывает нарисованное пользователем изображение и сравнивает его с первоначальным.
6. Система выводит на экран оценку схожести изображения в процентах и заносит эти данные в статистику.
7. Пользователь знакомится с оценкой и возвращается в главное меню.

### [ Сценарий 5 - Рисование на бумаге ]

1. Система выводит небольшое сообщение с рекомендациями для правильной работы программы. Также на экране присутствует кнопка окончания рисования.
2. Пользователь, воспроизводит ранее данное ему изображение на бумаге.
3. Закончив, пользователь нажимает на кнопку завершения рисования.
4. Система спрашивает у пользователя, действительно ли он хочет завершить рисование.
5. Если ответ пользователя отрицательный, сценарий возвращается к Шагу 2. В противном случае пользователю предлагается сделать фото его рисунка с помощью встроенной камеры устройства.
6. Пользователь делает фото рисунка.
7. Система выводит фото на экран и просит подтверждения правильности сделанного фото.
8. В случае отрицательного ответа пользователя, сценарий возвращается к шагу 6. В случае положительного ответа, система обрабатывает нарисованное пользователем изображение и сравнивает его с образцом.
9. Система выводит на экран оценку схожести изображений в процентах и заносит эти данные в статистику.
10. Пользователь знакомится с оценкой и возвращается в главное меню.

### [ Сценарий 6 - Просмотр истории и удаление её элементов ]

1. В главном меню пользователь выбирает элемент "История".
2. Система выводит на экран предыдущие раунды пользователя в хронологическом порядке.
3. Пользователь может по желанию удалить все раунды для освобождения памяти.
4. Если пользователь нажал на кнопку удаления всех раундов, система просит пользователя подтвердить действие. После подтверждения система удаляет историю.
5. Пользователь выбирает интересующий его раунд.
6. Система выводит на экран дату раунда, уровень сложности, начальное изображение и рисунок пользователя. Все данные, разумеется, соответствуют выбранному пользователем раунду.
7. Пользователь по желанию может удалить выбранный раунд для освобождения памяти.
8. Если пользователь нажал на кнопку удаления раунда, система просит пользователя подтвердить действие. После подтверждения система удаляет раунд.
9. Пользователь также может сообщить создателю о неверной оценке его рисунка. В таком случае он нажимает на соответствующую кнопку.
10. Если пользователь решил отправить сообщение о неверной оценке, система отправляет создателю изначальный рисунок, рисунок пользователя и оценку. Также система не будет учитывать этот раунд в статистике.

### [ Сценарий 7 - Рассказать друзьям ]

1. В главном меню пользователь выбирает элемент "История".
2. Система выводит на экран предыдущие раунды пользователя в хронологическом порядке.
3. Пользователь выбирает интересующий его раунд.
4. Система выводит на экран дату раунда, уровень сложности, начальное изображение и рисунок пользователя. Все данные, разумеется, соответствуют выбранному пользователем раунду.
5. Пользователь выбирает элемент "рассказать друзьям".
6. Пользователь выбирает социальную сеть, в которой он хочет поделиться с друзьями успехом в раунде.
7. Пользователь вводит логин и пароль от своего аккаунта в выбранной социальной сети.
8. Система выводит на экран текст и изображение поста, который будет опубликован на странице в соц. сети и требует у пользователя подтверждения.
9. После подтверждения со стороны пользователя система публикует пост.

### [ Сценарий 8 - Просмотр статистики ]

1. В главном меню пользователь выбирает элемент "Статистика".
2. Система выводит на экран отдельно график всех оценок рисунков пользователя на экране, рисунков пользователя на бумаге и объединённый график, на котором учитываются все рисунки, при этом рисунки на бумаге имеют больший вес, чем рисунки на экране. 
3. Пользователь изучает статистику и возвращается в главное меню.
